What is a Programming Language?

A Programming Language is a set of different categories of written symbols that instruct computer hardware to perform specified operations required by the designer.

  • Programming Language Skill: Knowing one or more languages to talk to and instruct the machine.
  • Problem Solving Skill: Identifying real-world problems and creating structured solutions.
  • Algorithm Development: Creating a clear, simple sequence of human-understandable instructions (steps) independent of any specific programming language.
  • Syntax: Sets of rules that govern the symbols used in a programming language. They determine how the programmer can make the computer perform specific operations.

Generations of Programming Languages

Languages evolved over 5 generations. The higher the generation, the easier it is to understand and use.

Low-Level Languages

Machine dependent.

  • 1st Gen (Machine - 1940s): Difficult to write. Dependent on hardware. Powerful resource utilization.
  • 2nd Gen (Assembly - 1950s): Uses symbolic names. Translated to machine code by an Assembler. Not portable.

High-Level Languages

Machine independent (portable).

  • 3rd Gen (1950s-70s): Procedural English-like syntax (math symbols like Z = A + B). Needs translation.
  • 4th Gen (Since late 70s): Non-procedural (SQL). Focuses on "what" needs to be done with minimal syntax rules.
  • 5th Gen (1990s): Natural languages used for AI and expert systems. Resembles human speech but requires powerful hardware.

Overview of Computer Organization

Every computer is divided into 6 functional logical units:

  • Input Unit: Obtains information from devices (keyboard, camera).
  • Output Unit: Places processed info on devices (monitor, speaker).
  • Memory Unit: Retains data immediately needed. Fast but temporary.
  • CPU (Central Processing Unit): The coordinator responsible for supervising all operations.
  • ALU (Arithmetic Logic Unit): Part of the CPU performing calculations and comparisons.
  • Secondary Storage: Disks holding saved data permanently.

Problem Solving & Flow Charts

A problem is an undesirable situation that prevents achieving a goal. Problem solving transforms the description of a problem into a solution using strategies.

  • Top-Down Design: Disintegrate a larger problem into smaller, manageable pieces (narrow the domain).
  • Bottom-Up Design: Build lowest level components first and assemble the whole process upward.

Algorithms & Flowcharts

An algorithm must be precise, simple, correct, and efficient. A flowchart is a graphical way to express these algorithm steps using standardized symbols (Ovals for Terminal, Rectangles for Process, Diamonds for Decision, Parallelograms for Input/Output).

Operators in Programming

An operator is a symbol that makes the machine take an action on operands (values).

  • Assignment (=): Causes the left operand to change its value to the right side (e.g., x = 3).
  • Compound Assignment (+=, -=, *=): Combines arithmetic with assignment. a -= 5 means a = a - 5.
  • Arithmetic (+, -, *, /, %): Standard math. Modulo (%) returns the remainder (e.g., 10 / 3 gives 3 remainder 1, so 10 % 3 = 1).
  • Relational (==, !=, >, <, >=, <=): Evaluates comparisons, producing a boolean value (True/False or 1/0).
  • Logical (!, &&, ||): Logical NOT (!) negates. Logical AND (&&) requires both operands to be true. Logical OR (||) requires one to be true.
  • Increment/Decrement (++, --): Adds or subtracts 1 from a variable. Prefix (++age) evaluated before assignment. Postfix (age++) evaluated after.
  • Conditional (?:): Ternary operation. condition ? true_result : false_result.
  • Comma (,): Combines multiple expressions. E.g. min = (m < n ? (mCount++, m) : (nCount++, n))
  • sizeof(): Calculates size of any data type in bytes. e.g. sizeof(int).
  • Type Casting: Explicitly converts data types. float f=3.14; int i = (int)f;

#include <iostream>

Mechanics of Creating a C++ Program

C++ programs typically go through five phases to be executed:

  • Edit: Programmer types C++ statements with an editor (stores as a .cpp file).
  • Preprocess: A preprocessor program executes automatically before compiling (commands like #include).
  • Compile: Validates and translates source code into machine/object code. Output is an incomplete executable (missing library codes).
  • Link: A linker connects object code with standard libraries to produce a fully executable image.
  • Load: The loader takes the executable and transfers it to memory (RAM) where the CPU finally runs it.

Compilers vs Interpreters & Paradigms

Translators take code not written in Machine Language and convert it.

Feature Compiler Interpreter
TranslationWhole program at onceLine-by-line
ExecutionFastSlower
OutputExecutable fileNo separate executable

Programming Paradigms

  • Procedural: Sequential instructions based on procedure calls. Easy to follow flow.
  • Structured: Writing program in small independent modules (functions) to ease testing.
  • Object-Oriented (OOP): Building programs from a collection of interactive Objects (Data + Actions) instead of a top-to-bottom list. Highly flexible.
