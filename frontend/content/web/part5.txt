JavaScript — Session 1: syntax, functions & modules

JavaScript is a programming language that runs in browsers and in other environments (e.g. Node.js). These notes focus on browser-oriented basics: enough syntax to read and write small scripts before the DOM session.

Strict mode
'use strict'; at the top of a file or function catches common mistakes (accidental globals, duplicate parameter names in old forms). Modules are strict by default.

Variables
• let — block-scoped, can be reassigned.
• const — block-scoped, binding cannot be reassigned (object contents can still change).
• Avoid var in new code (function-scoped, hoisting quirks).

Types (informal)
Primitives: string, number, bigint, boolean, undefined, null, symbol. typeof helps inspect; remember typeof null === "object" is a long-standing quirk. Everything else is an object (including arrays and functions).

Operators
Arithmetic: + - * / % **. Comparison: === and !== (prefer over ==). Logical: && || !. Optional chaining: obj?.prop?.nested. Nullish coalescing: a ?? b when you only want to fall back for null/undefined.

Strings
Template literals: `Hello, ${name}!` — multiline and interpolation. Methods: slice, includes, split, trim.

Functions
function add(a, b) { return a + b; }
const add2 = (a, b) => a + b;

Arrow functions have lexical this (inherit from surrounding scope) — important when passing callbacks. For methods that need their own this, a classic function or explicit binding may be better.

Scope
Block scope with let/const. Closures: inner functions remember variables from outer scopes — useful for factories and event handlers.

Arrays and objects
const items = [1, 2, 3]; items.map(x => x * 2); filter, reduce, find. Objects: shorthand { name }, computed keys, spread { ...obj }.

Control flow
if / else, switch (use break), for, for...of (arrays), for...in (object keys — careful with inherited properties). try / catch / finally for errors.

Modules
In browsers, use <script type="module" src="app.js">. import { x } from './utils.js'; export function helper() {}. Modules are deferred and run in strict mode. Keep files focused; re-export for public API surfaces.

Loading scripts
defer on classic scripts runs after HTML parse, in order. type="module" is deferred by default. Place scripts before </body> or use defer in <head> for performance.

DevTools quick tip
Console: log values, breakpoints in the Sources panel, watch expressions. Learn “step over” vs “step into” to trace logic.

Practice ideas
1. Write a function minutesToHHMM(n) returning a string like "1:05".
2. Given an array of numbers, return the sum using reduce.
3. Split app.js into math.js (exports) and main.js (imports) with type="module".

Key terms: let, const, template literal, arrow function, closure, module, defer.
